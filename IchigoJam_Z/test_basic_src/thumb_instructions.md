# Thumb 命令バイト早見表（IchigoJam USR用）

Cortex-M0+ は Thumb 命令のみ。命令は 2 バイト、リトルエンディアン（低バイトを先に POKE）。

---

## USR() 呼び出し規約

```
USR(ad, n)
  ad : BASIC 仮想アドレス（マシン語の先頭、#700〜）
  n  : 引数（R0 で渡される）
  戻り値: R0 の値
```

### レジスタ割り当て

| レジスタ | 値 | 用途 |
|---|---|---|
| R0 | `n`（引数） | 戻り値もここ |
| R1 | `RAM_AREA - #700` | BASIC 仮想アドレス → 実アドレス変換の基点 |
| R2 | `CHAR_PATTERN` | フォント ROM 先頭（文字パターン読み取りに使用） |
| R3 | `__aeabi_uidivmod` | ソフトウェア除算ヘルパ（Cortex-M0 は HW 除算なし） |

### アドレス変換

```
実アドレス = R1 + BASIC仮想アドレス
           = (RAM_AREA - #700) + ad
           = RAM_AREA + (ad - #700)
```

例: 変数 A (#8CC) を読む場合
```
実アドレス = R1 + #8CC = RAM_AREA - #700 + #8CC = RAM_AREA + #1CC
```

### Thumb モード

`IJB_usr()` 内部で `ad` の bit0 を 1 にして呼ぶ（Thumb フラグ）。
マシン語側では何もしなくてよい。`BX LR` で返るだけ。

## BASIC 仮想アドレスマップ

```
仮想アドレス  サイズ  内容
#000-#6FF     0x700   文字パターン ROM（読み取りは R2 経由）
#700-#7FF     0x100   PCG RAM（ユーザー定義フォント、マシン語置き場として最適）
#800-#8CB     0x0CC   @() 配列変数 [0]〜[101]（各 int16、102 要素）
#8CC-#9FF     0x134   変数 A〜Z（各 int16、26 変数）
#900-#BFF     0x300   VRAM（32×24 画面、シリアル版では未使用）
#C00-#FFF     0x400   BASIC プログラムリスト
#1000-#1001   0x002   リスト終端（常に 0）
#1002         0x001   キー状態（bit: space/↓/↑/→/←）
#1003         0x001   キーバッファ内文字数
#1004-#1081   0x07E   キーバッファ（126 文字）
#1082-#1149   0x0C8   ラインバッファ（200 バイト）
#114A-#117F   0x036   I2C バッファ（54 バイト、IoT コマンド用）
```

### 変数アドレス早見表

```
変数   BASIC 仮想アドレス   RAM_AREA オフセット
A      #8CC                 +#1CC
B      #8CE                 +#1CE
C      #8D0                 +#1D0
...
Z      #8FE                 +#1FE
```

計算式: `変数 X のアドレス = #8CC + (X - 'A') × 2`

### マシン語を置く場所の選択

| エリア | BASIC 仮想 | 特徴 |
|---|---|---|
| PCG RAM | `#700`〜`#7FF` | **推奨**。NEW / SAVE / LOAD で消えない。256 バイト |
| LIST エリア | `#C00`〜 | プログラム入力・NEW で上書きされる |
| @() 配列 | `#800`〜`#8CB` | BASIC から読み書きされるので競合に注意 |

## よく使う命令

| POKE バイト列 | Thumb 命令 | 動作 |
|---|---|---|
| `#70 #47` | `BX LR` | 関数から戻る（必ず末尾に置く） |
| `#2A #20` | `MOVS R0, #42` | R0 ← 42（定数代入の例） |
| `#NN #20` | `MOVS R0, #NN` | R0 ← NN（0〜255 の定数代入） |
| `#01 #30` | `ADDS R0, #1` | R0 ← R0 + 1 |
| `#NN #30` | `ADDS R0, #NN` | R0 ← R0 + NN |
| `#08 #5A` | `LDRH R0, [R1, R0]` | R0 ← MEM16[R1+R0]（BASIC RAM 読み込み、16bit） |
| `#08 #78` | `LDRB R0, [R1, R0]` | R0 ← MEM8[R1+R0]（BASIC RAM 読み込み、8bit） |
| `#08 #52` | `STRH R0, [R1, R2]` | MEM16[R1+R2] ← R0（BASIC RAM 書き込み、16bit） |

---

## LDRH エンコード詳細

`LDRH Rt, [Rn, Rm]` の Thumb T1 エンコード:

```
ビット: 15    9  8   6  5   3  2   0
        0101 101  Rm    Rn    Rt
```

| 命令 | Rm | Rn | Rt | 16進 | POKE バイト |
|---|---|---|---|---|---|
| `LDRH R0, [R1, R0]` | R0=000 | R1=001 | R0=000 | `0x5A08` | `#08 #5A` |
| `LDRH R1, [R1, R0]` | R0=000 | R1=001 | R1=001 | `0x5A09` | `#09 #5A` |
| `LDRH R0, [R1, R2]` | R2=010 | R1=001 | R0=000 | `0x5A48` | `#48 #5A` ← R2=ROM に注意 |

> **誤用注意**: `#48 #5A` は `LDRH R0, [R1, R2]` で R2（= CHAR_PATTERN ポインタ）が  
> オフセットになる。BASIC RAM を読むには `#08 #5A` を使うこと。

## テスト用スニペット

```basic
' Test 1: 定数 42 を返す
' MOVS R0,#42 = #2A #20 / BX LR = #70 #47
POKE #700,#2A,#20,#70,#47
PRINT USR(#700,0)
' Expected: 42

' Test 2: 引数をそのまま返す
' BX LR = #70 #47
POKE #700,#70,#47
PRINT USR(#700,99)
' Expected: 99

' Test 3: 引数 + 1 を返す
' ADDS R0,#1 = #01 #30 / BX LR = #70 #47
POKE #700,#01,#30,#70,#47
PRINT USR(#700,10)
' Expected: 11

' Test 4: 変数 A の値を読む（RAM アクセス確認）
' LDRH R0,[R1,R0] = #08 #5A / BX LR = #70 #47
A=123
POKE #700,#08,#5A,#70,#47
PRINT USR(#700,#8CC)
' Expected: 123
```
